Exact Deduplication

Quality
Quality Filters

Toxicity
Toxicity Masking

Byte-Pair
Byte-Pair Encoding
Algorithmically compressing frequent word combinations into single optimal token IDs.
Subword
Subword Chunks
Splitting entirely unrecognized words into distinct morphological syllables rather than crashing.
Vocab
Vocab Escalation

Topic 1 — token + positional embedding
Integer token IDs are looked up in a giant table to retrieve dense vectors. Positional information is injected so the transformer knows word order.
Lookup
Lookup Matrix
Translating distinct integer Token IDs into multi-dimensional floating point vectors.
Dense
Dense Representation
Encoding linguistic traits geometrically. Words with similar meanings cluster together spatially.
Lexical
Lexical Ambiguity
Why basic embeddings fail to physically separate contextual differences like 'River Bank' vs 'Bank Vault'.
The
The Order Problem
Transformers process all vectors simultaneously as an unordered set, leaving them entirely sequence-blind.
Absolute
Absolute Encodings
Using mathematically static Sine and Cosine waves clamped to integer timestamps (The 2017 standard).
Relative
Relative Distances
Learning the specific positional distance between two tokens, rather than relying on absolute grid timestamps.
RoPE
RoPE Mechanics
Rotary Positional Embeddings uniquely rotate vectors mathematically in coordinate space to perfectly encode relative overlap.
VRAM
VRAM Loading
Why huge embedding matrices are often parallel sharded across multiple GPUs to prevent memory exhaustion.

Topic 2 — embedding algorithms
How the embedding vectors in Layer 1 are trained. A progression from shallow count-based methods to deep contextual transformers.
One-Hot
One-Hot Vectors
The primitive approach. A vocabulary of 50K requires a 50K array of zeros with a single 1. Causes matrix bloat.
Count-Based
Count-Based TF-IDF
Term Frequency tracking. Evaluates global word importance but entirely destroys grammatical word locality.
Word2Vec
Word2Vec
The breakthrough shallow neural network that mapped vectors securely by training them to predict local adjacent context windows.
CBOW
CBOW Paradigm
Continuous Bag of Words: Given the surrounding context words, correctly predict the missing center word.
Skip-Gram
Skip-Gram
The inverse of CBOW: Given a single center word, algorithmically predict all the surrounding context words.
GloVe
GloVe Matrices
Global Vectors for Word Representation. Bypasses local context sliding windows by efficiently factorizing global co-occurrence matrices.
FastText
FastText
Further decomposes vocabulary words into literal n-gram character chunks, unlocking the ability to embed totally unseen, out-of-vocabulary words.
ELMo
ELMo Context
The final pre-transformer technique utilizing deep bi-directional LSTMs to evaluate contextual vector shifting.

Topic 3 — attention mechanism
The heart of the transformer. Every token computes relationships with every other token via query, key, and value projections. The K and V vectors computed here are what the KV cache (Layer 7) stores and reuses at inference time.
QKV
QKV Projections
Each token duplicates itself by linearly multiplying into three distinct states: The Query, the Key, and the Value.
Dot-Product
Dot-Product Match
The model mathematically multiplies every Query against every single previous Key to evaluate semantic relevance.
Scaling
Scaling Factor
Dividing the massive dot-product integer results by the square root of dimensions to prevent gradient explosions.
Causal
Causal Masking
Applying an infinite negative value to all diagonal future tokens to physically block the autoregressor from cheating.
Softmax
Softmax Filter
Squashing the raw attention scores into a perfect localized probability distribution from zero to one.
Contextual
Contextual Fusion
Multiplying the normalized attention distribution directly against the Value arrays to blend relevant context.
Multi-Head
Multi-Head Slicing
Splitting the massive dimension arrays into parallel 'heads' so the model can track multiple different grammar relationships simultaneously.
GQA
GQA Efficiency
Group Query Attention aggressively shares Keys and Values across Heads to dramatically drop VRAM cache requirements.

Topic 4 — transformer block internals
After attention, each token passes through a feed-forward network. Layer normalisation and residual connections stabilise training across many stacked blocks.
Residual
Residual Skips
Connecting input pathways directly past the heavy calculation blocks. This is vital to prevent mathematical signals from dissolving in deep networks.
Layer
Layer Normalization
Normalizing vectors uniformly. The industry decisively moved from explicit Post-Norm to mathematically stable Pre-Norm configurations.
RMSNorm
RMSNorm
Root Mean Square Normalization strips explicit mean-centering out of the equations, resulting in faster math without accuracy loss.
Factual
Factual Memory (FFN)
While Attention routes the active context, the dense Feed Forward block operates as the core internalized factual knowledge bank.
SwiGLU
SwiGLU Activations
The modern baseline activation replacement for RELU, utilizing smoothly gated linear units for faster signal convergence.
Dimension
Dimension Expansion
The FFN routinely expands the token dimensionality physically by 4x, aggressively cross-referencing sub-features, before projecting it back down.
Architectural
Architectural Stacking
Stringing up to 100+ of these exact sequential modular transformer blocks back-to-back to force immense cognitive depth.
Gradient
Gradient Highways
Because of ResNets, backwards-flowing gradients during backpropagation safely travel all the way down to Layer 0 without vanishing.

Topic 5 — architecture families
How many transformer blocks, in what configuration, with what training objective — these decisions define whether you get BERT, T5, or GPT.
Recurrent
Recurrent LSTMs
The archaic baseline. Relied on sequential memory hidden-states that degraded over long distances and throttled parallelization.
Bidirectional
Bidirectional BERT
Focuses on reading the entire sequence forwards and backwards simultaneously. Supremely powerful at structural classification and search mapping.
Masked
Masked Language Task
Training Encoders by actively blacking out 15% of the text and forcing the model to contextually guess the missing gaps.
Encoder-Decoder
Encoder-Decoder
Initial architecture splitting reading and writing. Powerful for explicit tasks like Language Translation but suffered scaling friction.
Decoder-Only
Decoder-Only
The current industry monolith. Reaping the benefits of strictly masked autoregression to scale infinitely without architectural fractures.
Next-Token
Next-Token Reality
The mathematically pure objective: entirely discard bidirectional reads and simply predict what comes next. Scale unlocks emergence.
Mixture
Mixture of Experts
MoE replaces a monolithic dense FFN with multiple sub-networks. A router selectively picks 2 'experts' per token, maintaining speed while inflating raw params.
Scaling
Scaling Laws
The empirical discovery that compute, data, and parameter counts scale capability almost linearly on a log-log curve.

Topic 6 — output head
After N transformer blocks, the final hidden state is projected to vocabulary logits and a probability is sampled for the next token. This closes the forward pass and triggers the inference machinery of Layer 7.
The
The Linear Head
The massive finale matrix. Projects the internal structural hidden state array forcefully outwards back into the exact size of the vocabulary.
Logits
Logits
The raw, unnormalized floats mapped to every single word in the dictionary. Extreme peaks indicate certainty.
Softmax
Softmax Normalization
The brutal mathematical function that violently converts logits into a flawless probability distribution where all values equal exactly 1.0.
Temperature
Temperature Scaling
Dividing logits prior to Softmax. A value of 0.1 locks deterministic paths, while 1.5 flattens probability layers causing chaotic creativity.
Top-K
Top-K Slicing
Immediately pruning the distribution to only sample from the absolute top 50 highly-probable words regardless of their distribution tail.
Top-P
Top-P Slicing
Nucleus sampling limits the pool to the minimum number of words whose probabilities sum up to a threshold like 0.95, dynamically adjusting to certainty.
Frequency
Frequency Penalities
Logit biasing parameters that actively subtract mathematically from words that have already appeared frequently in the generation.
Stop
Stop Tokens
Specialized End-of-Text characters injected into the training set. When sampled, the inference engine abruptly severs the connection.

Topic 7 — inference & deployment
The missing chapter in most LLM courses. Training happens once — inference runs billions of times. Over 90% of the total cost of running an LLM is inference. This layer covers the key techniques that make production deployment fast, cheap, and scalable.
The
The Memory Wall
Compute is vastly faster than RAM. Inference generation is fundamentally bottlenecked by VRAM memory-bandwidth, not raw floating-point operations.
Prefill
Prefill Parallelism
The first step of inference. Taking the entire prompt and blasting it aggressively through parallel matrix multiplications simultaneously.
Serial
Serial Decoding
The execution loop. Autoregressively computing generation frame-by-frame, violently starved by the inability to math in parallel.
The
The KV Cache
Massive VRAM overhead. Physically caching all historical Key and Value vectors for the entire context window so they don't have to be recalculated perfectly.
PagedAttention
PagedAttention
The vLLM revelation. Dynamically allocating non-contiguous memory blocks on the GPU to store the KV Cache perfectly, eliminating VRAM fragmentation.
FlashAttention
FlashAttention
A brutal hardware-aware kernel that fuses operations directly into ultra-fast SRAM caches, dramatically cutting slow HBM Round-Trips.
Model
Model Quantization
Mathematically dropping weights from 16-bit to 4-bit primitives using frameworks like AWQ, actively doubling inference speed on strict bandwidth bounds.
Speculative
Speculative Decoding
Utilizing a tiny, fast 7B 'draft' model to blindly guess 5 future tokens, submitting them to the 70B model for 1-step parallel verification. Massive speedups.

Topic 8 — pretraining
Training the base model from scratch on trillions of tokens. The most expensive stage — but everything downstream depends on the quality of this foundation.
Weight
Weight Initialization
Using strict stochastic algorithms like Kaiming or Xavier to initialize parameters explicitly to prevent immediate catastrophic mathematical collapse.
Gradient
Gradient Descent
The core concept of computationally navigating multi-dimensional calculus topography in reverse to map the steepest path to zero loss.
Cross-Entropy
Cross-Entropy Loss
Comparing the generated soft probability curve aggressively against the rigid One-Hot target truth vector to calculate numerical error.
Backpropagation
Backpropagation
Using the explicit Chain Rule of Calculus to forcefully pass the calculated error curve backwards across every single matrix in the transformer block.
The
The Epoch Ceiling
Unlike basic Vision models, LLMs undergo catastrophic degradation and memorization loops if they observe the exact same pretraining text more than twice.
Emergence
Emergence Grokking
The phenomenon where massive network parameters violently snap into geometric alignment after massive iteration scales, spontaneously unlocking unprogrammed logic.
Trillion
Trillion Token Scale
Llama 3 absorbed over 15 Trillion tokens of data, definitively proving that massive parameter counts are deeply under-trained without immense datasets.
Perplexity
Perplexity Physics
The primary evaluation mathematical metric. A lower perplexity indicates the network's probability distributions are explicitly converging with reality.

Topic 9 — training optimizations
Engineering solutions that make large-scale training stable, fast, and memory-efficient. Every frontier training run uses all of these simultaneously.
AdamW
AdamW Dynamics
The undisputed standard optimizer. Aggressively separates Weight Decay from the primary gradient updates to stop the model from rapidly forgetting structures.
Moving
Moving Momentum
The optimizer forcefully tracks exponential moving averages of past gradients (Beta 1), allowing the optimization 'ball' to seamlessly skip over fake local valleys.
Cosine
Cosine Learning Rates
Gradually decaying the intensity of updates following a Cosine wave. A brutal drop-off in the final 5% of training is crucial for fine-detail convergence.
Linear
Linear Warmups
Starting the learning rate functionally at zero and ramping it rapidly across the first 2,000 steps to prevent instant matrix explosion on unaligned variables.
Mixed
Mixed Precision BF16
Running the heavy forward and backward passes using stripped-down BFloat16 precision, but carefully logging gradients back into FP32 Master Weights.
Data
Data Parallelism
Duplicating the entire Model across multiple GPU nodes. Each GPU trains intensely on a different slice of data, averaging gradients at the end of the microstep.
Tensor
Tensor Parallelism
When a model is too big for a single GPU, massive linear matrices are violently sharded horizontally and vertically across interconnected NVLink pipelines.
ZeRO
ZeRO Redundancy
Microsoft's DeepSpeed integration. Eliminating duplicate Optimizer States and Gradient tracking across nodes to massively decrease multi-node VRAM requirements.

Topic 10 — supervised fine-tuning & PEFT
Adapting the base model to follow instructions and domain tasks. LoRA and QLoRA make this feasible on consumer hardware by training only 1% of parameters.
The
The Formatting Shift
Pretraining teaches text continuation. SFT (Supervised Fine Tuning) structurally teaches the model that it must definitively format completions as conversational Answers.
Instruction
Instruction Masking
During backpropagation, loss is explicitly 'masked' on the Prompt tokens. We ONLY grade and penalize the model for its explicit generated Response logic.
ChatML
ChatML System Prompts
Embedding specialized `<|im_start|>` meta-tokens to rigorously define boundaries between the immutable System directive and the volatile User prompt.
Full-Parameter
Full-Parameter VRAM
Standard Fine-Tuning forces gradient updates natively across all 70B weights. This requires massive clustered H100s to hold the optimizer states.
Low-Rank
Low-Rank Adaptation
LoRA explicitly freezes the base brain. It trains two tiny, microscopic matrices (A and B) that inject calculated gradient deviations alongside the frozen weights.
QLoRA
QLoRA Int4 Stacking
Quantizing the heavy frozen brain aggressively down to 4-bits, leaving only the tiny LoRA adapters in 16-bit. Unlocks 70B fine-tuning securely on desktop grids.
Rank
Rank & Alpha Tuning
The 'R' establishes how thick the microscopic matrices are. The 'Alpha' multiplier ensures the signal broadcasts loud enough to overpower the frozen brain.
Merged
Merged Inferences
At deployment, the tiny calculated LoRA matrix parameters are violently summed directly into the base weights, completely eliminating inference latency overhead.

Topic 11 — alignment (RLHF)
Turning a text predictor into a helpful, honest, harmless assistant. The reward model encodes human preferences; reinforcement learning optimises the policy against it.
Alignment
Alignment Dilemma
The constant tug-of-war. Forcing an AI to be aggressively 'Harmless' fundamentally destroys its ability to be computationally 'Helpful' without strict calibration.
Human
Human Preference Grids
Hiring armies of human annotators to strictly evaluate two parallel model outputs (A vs B) and flag the explicitly superior formatting and logic.
The
The Reward Model
Training an entirely separate LLM to read the (A vs B) comparisons and learn to natively output a scalar float 'Score' mapping human preference logic.
PPO
PPO Policy Gradients
Proximal Policy Optimization. The intense reinforcement loop where the base generator desperately adjusts its output distribution solely to maximize the Reward Model's scalar score.
KL
KL Divergence
A mathematical leash constraint explicitly locking the PPO model from drifting violently away from its pretrained statistical understanding just to hack high scores.
DPO
DPO Optimization
Direct Preference Optimization. An algorithmic revelation that entirely removes the separate Reward Model, utilizing binary cross-entropy to train PPO logic directly into the base weights.
Constitutional
Constitutional Prompts
Anthropic's methodology. Providing a high-end LLM an explicitly written Constitution to auto-generate and self-align its own synthetic preference data securely.
Jailbreak
Jailbreak Collapse
The unfortunate byproduct of soft-alignment. Creative role-play formatting or base64 overrides frequently bypass the superficial safety boundary mechanisms.

Topic 12 — reasoning training (RLVR)
Teaching models to generate extended chain-of-thought reasoning using verifiable rewards instead of learned reward models.
System
System 2 Reasoning
Shifting the AI paradigm from intuitive fast-twitch generation (System 1) into explicit, heavily token-consuming logical trace pathways (System 2).
Chain-of-Thought
Chain-of-Thought
Forcing the model to explicitly write out step-by-step calculations. This physically unlocks hidden sequential logic states by consuming computational token time.
Test-Time
Test-Time Scaling
The massive revelation that allowing a model to generate 10x more intermediate tokens prior to answering scales accuracy faster than 10x larger model weights.
GRPO
GRPO Framework
Group Relative Policy Optimization. Generating 8 variations of an answer and dynamically rewarding the versions that logically deviate towards correctness without a separate value model.
Verifiable
Verifiable Rewards
Completely overriding subjective human Reward Models with explicit Python Unit-Tests and strict math heuristics that return definitive binary Truth feedback.
Eliminating
Eliminating the Judge
By relying on explicit mathematical truth rather than vague human 'Helpfulness', RLVR scales aggressively into unmapped super-human logic discovery.
Majority
Majority Voting
Taking the 10 parallel generated responses from explicit test-time compute drops and aggressively filtering them securely using Consensus logic.
DeepSeek
DeepSeek Architectures
The pinnacle of reinforcement logic. Aggressively scaling pure GRPO across mathematically verifiable tasks until the model spontaneously learns to strictly self-correct errors.

Topic 13.5 — prompt engineering & RAG NEW
Models are frozen in time after training. To give them fresh data without expensive fine-tuning, we use Retrieval-Augmented Generation (RAG) to dynamically fetch and inject facts into their prompt context before they generate.
Context
The Context Window
The model's short-term memory during a single conversation. Raging from 8K tokens (Llama 3 8B) to 2M tokens (Gemini 1.5 Pro). Extremely expensive computationally to fill completely due to quadratic attention scaling.
Ingestion
Data Ingestion
To feed data into an LLM, raw files (PDFs, docs, unstructured web scrapes) are systematically parsed and cleaned via ETL pipelines into pure text before processing.
Chunking
Semantic Chunking
Deep documents cannot be searched effectively en masse. They are algorithmically sliced into 500-token chunks with a 50-token semantic overlap to ensure critical context isn't severed in half.
Embed
Vectorization
Each individual chunk is mathematically passed through a distinct Embedding Model (like text-embedding-3) which translates the human linguistics into a dense spatial array of floats (e.g. [0.15, -0.62...]).
Hybrid
Hybrid Search
Standard vector cosine similarity fails at exact keyword matching (like ID JIRA-712). Hybrid Search algorithms pair Vectors heavily with traditional keyword-based BM25 to achieve optimal retrieval accuracy.
Rerank
Reranking Encoders
Because Vector DBs only perform mathematically approximate searches for speed, a heavy Cross-Encoder AI is often deployed to slowly and intimately re-sort the Top-50 results to pick out the absolute Top-5 golden documents.
GraphRAG
Knowledge Graphs
A cutting-edge technique where vector data is additionally mapped out onto vast nodal relationship structures. This solves holistic queries like "Who works for who?" simply by tracing the explicitly coded edges.
RAG
Retrieval workflow
User queries system → System queries Vector DB → DB returns Top-K documents → System pastes documents + user query into a giant prompt → LLM reads it all natively and generates correct answer.
