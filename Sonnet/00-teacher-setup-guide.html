Option A — Google Sites (recommended for most schools)

This is the smoothest path if your school uses Google Workspace. Students access everything through their school Google account with no extra logins.

1

Create a new Google Site

Go to sites.google.com and click the + button to create a new site. Give it a name like "LLM Curriculum" and choose a simple template.

2

Add a page for each widget

In the Pages panel on the right, click + New page and name it (e.g. "Layer Map", "Pretraining Pipeline", etc.). Create one page per HTML file.

3

Embed the HTML on each page

On each page, click Insert → Embed. Switch from "URL" to Embed code. Open the HTML file in a text editor, select all the text, copy it, and paste it into the embed box. Click Next → Insert.

The embed box has a character limit. If the HTML file is too large to paste, use the Google Drive method in Option B instead.

4

Resize the embed to fill the page

Click the embed block you just inserted, then drag the resize handle to make it taller. Aim for at least 700px height so the widgets display properly without internal scrollbars.

5

Publish the site

Click Publish in the top-right corner. Set visibility to Anyone at your organisation (or public if your school allows). Copy the published URL.

6

Add the link to Google Classroom

In Google Classroom, create a new Material or Assignment. Click Add → Link and paste the Google Sites URL. Students click the link and the full interactive curriculum opens in their browser.

Option B — Google Drive (simplest, no setup)

Upload the HTML files directly to Google Drive and share the links. This works immediately but requires students to download each file to view it — it does not open in-browser from Drive.

1

Upload files to Google Drive

Drag all 8 HTML files into a folder in Google Drive. Right-click the folder → Share → set to Anyone with the link can view.

2

Add files to Google Classroom

In Google Classroom, create a Material. Click Add → Google Drive and select each file. Students click the file → Download → open in browser. Works on all devices.

Tip: rename the files with short descriptive names before uploading — for example "Layer Map.html" and "Pretraining Pipeline.html" — so students can identify them easily in Classroom.

Option C — GitHub Pages (permanent public URL)

The most durable solution. Creates a permanent public URL that never expires and works anywhere in the world without a school account. Requires a free GitHub account.

1

Create a free GitHub account

Go to github.com and sign up. Free accounts include unlimited public repositories and GitHub Pages hosting.

2

Create a new repository

Click New repository. Name it llm-curriculum. Set it to Public. Check Add a README file. Click Create repository.

3

Upload the HTML files

In your new repository, click Add file → Upload files. Drag all 8 HTML files in. Rename 01-curriculum-layer-map.html to index.html so it becomes the home page. Click Commit changes.

4

Enable GitHub Pages

Go to Settings → Pages. Under Source, select Deploy from a branch. Choose main branch, / (root) folder. Click Save. Wait 2–3 minutes.

5

Copy your permanent URL

Your site is now live at https://yourusername.github.io/llm-curriculum/. Each file has its own URL: https://yourusername.github.io/llm-curriculum/02-pretraining-pipeline.html etc. Paste these into Google Classroom as link assignments.

Recommended classroom workflow

Start here

Week 1 — Layer Map

Use 01-curriculum-layer-map.html as the course syllabus. Show students the full layer stack on day one. Return to it before each new unit so students see where each topic fits in the bigger picture.

Foundations

Weeks 2–4 — Embedding & Attention

Work through files 05 (Embedding deep-dive) and 06 (CBOW/Skip-gram) to build intuition for vectors, then use file 07 (Architecture families) to show how attention combines everything.

Mid-course

Weeks 5–7 — Training

Use file 02 (Pretraining pipeline) to cover all four training stages: pretraining, training optimizations, SFT, and PEFT. This is the most technically dense unit — allocate extra time.

Advanced

Weeks 8–10 — Alignment & Reasoning

File 03 (RLHF and reasoning) covers PPO, DPO, GRPO, and the benchmarks used to evaluate reasoning models. Pairs well with current AI news about model releases.

Extension

Weeks 11–12 — Multimodal

File 04 (Vision & Speech) rounds out the course by showing how the transformer architecture extends to images and audio. A natural ending point connecting back to Layer 12 on the layer map.

Reference

Throughout — Paper annotations

Every topic card in the layer map (file 01) includes the original research paper with arXiv link. Encourage advanced students to read the abstracts. Many papers are surprisingly readable once students understand the concepts.

Technical notes for IT / administrators

All HTML files are completely self-contained. They have no external dependencies, make no network requests, store no data, and require no authentication. They are safe to host on any school web server or Google Sites instance.

The files use only standard HTML5, CSS3, and vanilla JavaScript. They have been tested and work correctly in Chrome, Firefox, Safari, and Edge on Windows, macOS, iOS, and Android. They work on Chromebooks via the Chrome browser.

There are no cookies, no tracking, no analytics, no third-party scripts, and no user data collection of any kind.

If your school's Content Security Policy blocks inline scripts, the interactive features will not work on a school-hosted server. GitHub Pages and Google Sites both allow inline scripts and are the recommended hosting options.

Getting help

If any of the files do not display correctly, the most common cause is that the browser is opening the file as plain text rather than rendering it as HTML. Make sure the file is opened with File → Open in the browser (not double-clicked to open in a text editor). The file extension must end in .html.

For Google Sites embed issues, try pasting the HTML into a plain text editor first to verify it copied correctly, then paste from the text editor into Google Sites.
