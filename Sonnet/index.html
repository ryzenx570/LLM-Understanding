Complete LLM curriculum — 16 layers
Every topic placed at the layer where it actually operates. Layer 7 (Inference & Deployment) and Layer 13 (Frameworks & Tooling) are new — inserted to bridge the gap between conceptual algorithms and real-world implementation. Click any layer or block to explore.
Layer 0 — input & data
Before a single weight is updated, raw text must be collected, cleaned, and converted into integer token IDs. Everything the model knows is determined by this layer.
Corpus
Data curation
Deduplication, quality filtering (fastText classifiers), domain balancing. Llama 3: 15T tokens. GPT-3: 500B tokens. Data quality beats data quantity at every scale.
BPE
Byte-pair encoding tokenisation
Merges most-frequent byte pairs until vocab size is reached. 50K–100K tokens. "cat" → 1 token. "neurotransmitter" → 4 tokens. 1–4 bytes per token depending on language and script.
Bytes
Token byte sizes
ASCII English: 1–3 bytes/token. Chinese: 3 bytes/char. Emoji: 4 bytes. Numbers often split digit-by-digit — the core reason LLMs struggle with arithmetic.
Scale
Chinchilla scaling laws
Hoffmann et al. 2022: optimal compute = ~20 tokens per parameter. Llama 3 deliberately over-trains for inference efficiency. GPT-3 was heavily undertrained by Chinchilla standards.
Scaling Laws for Neural Language Models — Kaplan et al., OpenAI 2020 (arXiv:2001.08361).
Training Compute-Optimal LLMs (Chinchilla) — Hoffmann et al., DeepMind 2022.
The Llama 3 Herd of Models — Meta 2024 (arXiv:2407.21783).
Layer 1 — token + positional embedding
Integer token IDs are looked up in a giant table to retrieve dense vectors. Positional information is injected so the transformer knows word order.
Lookup
Embedding table
One row per vocabulary item. Each row = a vector of 768 (GPT-2) to 12,288 (GPT-3) floats. GPT-3: 50,257 × 12,288 = 617M parameters just in the embedding layer.
Geometry
Vector as meaning
Similar words cluster nearby. king − man + woman ≈ queen. Context reshapes meaning — "bank" near "river" vs "bank" near "loan" produces different vectors.
Position
Positional encoding evolution
Sinusoidal (2017) → Learned absolute (BERT/GPT-2) → ALiBi → RoPE (rotates Q&K, dominant since 2022) → YaRN / LongRoPE (500K+ context).
RoPE
Rotary position embedding
Rotates Q and K vectors by angle proportional to position. Q·K dot product naturally encodes relative distance. Used in Llama 1/2/3, Mistral, Falcon, Gemma. Llama 3: 128K context.
RoFormer: Enhanced Transformer with Rotary Position Embedding — Su et al. 2021 (arXiv:2104.09864).
Layer 2 — embedding algorithms
How the embedding vectors in Layer 1 are trained. A progression from shallow count-based methods to deep contextual transformers.
CBOW
Continuous bag of words
Average context word vectors → predict the missing centre word. Many→one. Fast training, good for frequent words. Deep parallel: BERT's MLM is CBOW with a full transformer instead of a shallow average.
Skip-gram
Skip-gram + negative sampling
One word → predict all surrounding words. One→many. 4× more training signals per position. Excellent for rare words. Negative sampling updates only 5–20 wrong words per step — 1000× speedup.
GloVe
Global vectors (Stanford 2014)
Factorises the full corpus co-occurrence matrix globally. More stable training. Paris:France ≈ Berlin:Germany emerges from the global matrix structure.
FastText
Subword embeddings (Meta 2017)
Breaks words into character n-grams. Can represent unseen words. Essential for morphologically rich languages like Turkish, Finnish, and Arabic.
Efficient Estimation of Word Representations in Vector Space (Word2Vec) — Mikolov et al., Google 2013 (arXiv:1301.3781).
GloVe: Global Vectors for Word Representation — Pennington, Socher, Manning, Stanford 2014.
Layer 3 — attention mechanism
The heart of the transformer. Every token computes relationships with every other token via query, key, and value projections. The K and V vectors computed here are what the KV cache (Layer 7) stores and reuses at inference time.
QKV
Scaled dot-product attention
Q·Kᵀ / √d_k gives alignment scores. Softmax normalises to probabilities. Multiply by V for weighted sum of values. Multi-head: run in parallel with different learned projections to capture different relationship types.
Mask
Causal mask (decoder)
Upper-triangle of −∞ prevents attending to future tokens. After softmax, −∞ → exactly 0. At inference each new token reruns full attention — but the KV cache (Layer 7) stores all past K+V so only 1 new computation is needed per step.
Window
Sliding window attention
Each token attends only to W nearest neighbours → O(N·W) instead of O(N²). Mistral 7B: window=4,096 × 32 layers = 131K effective context. Rolling KV cache discards old key-value pairs automatically.
GQA
Grouped query attention
Multiple Q heads share one K+V head. Llama 3: 32 Q heads, 8 KV heads → 4× smaller KV cache at inference. FlashAttention computes the same result as full attention but tiles it on GPU for 2–4× memory savings.
Attention Is All You Need — Vaswani et al., Google Brain 2017 (arXiv:1706.03762). 90K+ citations.
FlashAttention-2 — Dao et al. 2022/2023. Standard in nearly every frontier model training and inference run.
Layer 4 — transformer block internals
After attention, each token passes through a feed-forward network. Layer normalisation and residual connections stabilise training across many stacked blocks.
FFN
Feed-forward network
Two linear layers with GELU activation between them, 4× wider than embedding dim. Applied to each token independently. Believed to store most of the model's factual knowledge.
Norm
Layer normalisation
Pre-norm (applied before each sublayer) is now standard — more stable than original post-norm. Rescales activations to mean 0, std 1. Prevents exploding/vanishing gradients across N stacked blocks.
Skip
Residual connections
Output = LayerNorm(input) + sublayer(input). The identity path bypasses each sublayer. Makes gradients flow directly from output to early layers — the critical ingredient enabling 96-layer depth in GPT-3.
Stack
Stacking blocks × N
GPT-2: 12 blocks. GPT-3: 96 blocks. Lower blocks learn syntax and surface patterns. Upper blocks encode reasoning, facts, and abstract concepts.
Layer 5 — architecture families
How many transformer blocks, in what configuration, with what training objective — these decisions define whether you get BERT, T5, or GPT.
Enc-only
BERT · RoBERTa · DeBERTa
Bidirectional attention — no causal mask. MLM + NSP pretraining. [CLS] for classification, [MASK] for prediction, [SEP] for segment boundary. Cannot generate text. Powers Google Search, classification, NER.
Enc-dec
T5 · BART · mT5
T5: text-to-text framing for every task. BART: 5 corruption strategies. Cross-attention connects encoder to decoder. Used in Whisper for speech recognition.
Dec-only
GPT · Llama · Claude · Gemini
Causal mask only. No encoder, no cross-attention. One objective: next-token prediction. Scales cleanly to trillions of parameters. All frontier assistants use decoder-only architecture + RLHF alignment on top.
Why dec wins
Why decoder-only dominates
One simple loss scales to any text. No encoder-decoder split. The prompt IS the context — decoder attends to it causally. RLHF converts the next-token predictor into an assistant without any architecture change.
BERT: Pre-training of Deep Bidirectional Transformers — Devlin et al., Google 2018 (arXiv:1810.04805).
Exploring the Limits of Transfer Learning (T5) — Raffel et al., Google 2019 (arXiv:1910.10683).
BART: Denoising Sequence-to-Sequence Pre-training — Lewis et al., Meta 2019 (arXiv:1910.13461).
Layer 6 — output head
After N transformer blocks, the final hidden state is projected to vocabulary logits and a probability is sampled for the next token. This closes the forward pass and triggers the inference machinery of Layer 7.
Logits
Linear projection
A weight matrix maps the D-dimensional hidden state to V-dimensional logits (one per vocab item). GPT-3: 12,288 → 50,257. This matrix shares weights with the embedding table (weight tying) in modern models.
Prob
Softmax + sampling
Softmax converts logits to probabilities summing to 1. Temperature <1 = sharper (more confident). Temperature >1 = flatter (more creative). Top-p: sample from smallest set of tokens summing to p (e.g. 0.9).
Loss
Cross-entropy loss
−log(probability assigned to correct next token). Averaged across all positions. Perplexity = exp(loss). Lower is better. GPT-2 on Wikipedia: ~35. GPT-4: ~5.
Decode
Decoding strategies
Greedy: always pick the highest-probability token. Beam search: keep top-k sequences at each step. Temperature + top-p for creative generation. Each decode step benefits from the KV cache in Layer 7.
Layer 7 — inference & deployment NEW
The missing chapter in most LLM courses. Training happens once — inference runs billions of times. Over 90% of the total cost of running an LLM is inference. This layer covers the key techniques that make production deployment fast, cheap, and scalable.
Phases
Prefill vs decode
Prefill: all prompt tokens processed in parallel in one forward pass — compute-bound. Builds the initial KV cache. The pause before streaming begins. Decode: output tokens generated one at a time — memory-bandwidth-bound. Output tokens cost 3–10× more than input tokens.
KV cache
Never recompute past attention
Stores Key and Value vectors for every past token at every layer. Each decode step reads cached K+V and computes only 1 new K+V — turning O(N²) into O(N). Llama 3 8B: ~2.5 MB per token. PagedAttention (vLLM 2023) reduces cache memory waste from 80% to <4%.
Quantization
Shrink weights: BF16 → INT4
Model weights in INT4 use ¼ the memory of BF16. Llama 3 70B: 140 GB (BF16) → 35 GB (INT4) — fits on one consumer GPU. Decode is memory-bandwidth-bound, so 4-bit weights are read up to 4× faster. GPTQ and AWQ achieve near-lossless quality at INT4.
Speculative decoding
Draft 4 tokens, verify in 1 step
A small draft model proposes k tokens. The large target model verifies all k in one parallel forward pass — same cost as generating 1 token normally. 70–90% acceptance rate on coding tasks yields 2–4× speedup. Standard in vLLM, SGLang, and TensorRT-LLM.
Batching
Continuous batching
Continuous (in-flight) batching inserts new requests the moment any sequence completes — keeping GPU utilisation above 95%. Cuts per-token cost by ~85% at 32 concurrent requests vs static batching.
Engines
vLLM · SGLang · TensorRT-LLM
vLLM: fastest path to production, PagedAttention, broad model support, 60s cold start. SGLang: best for shared-prefix workloads (RAG, chatbots). TensorRT-LLM: peak throughput after 28-min compilation, 13% faster than vLLM at high concurrency.
Efficient Memory Management for Large Language Model Serving with PagedAttention (vLLM) — Kwon et al., UC Berkeley / SOSP 2023. 24× throughput improvement. Introduced continuous batching + PagedAttention.
Fast Inference from Transformers via Speculative Decoding — Leviathan et al., Google 2023 (arXiv:2211.17192). Proved draft+verify produces identical output distribution to the target model.
AWQ: Activation-aware Weight Quantization for LLM Compression — Lin et al. 2024 (arXiv:2306.00978). Near-lossless INT4 quantization. Default method in vLLM.
GPTQ: Accurate Post-Training Quantization for Generative Pre-trained Transformers — Frantar et al. 2022 (arXiv:2210.17323). 4-bit quantization without retraining. Foundation of GGUF / llama.cpp.
Layer 8 — pretraining
Training the base model from scratch on trillions of tokens. The most expensive stage — but everything downstream depends on the quality of this foundation.
Objective
Next-token prediction
Predict each next token given all previous ones. One loss, any text, no labels required. At scale, this simple objective causes the model to learn grammar, facts, world knowledge, coding, and reasoning as side effects.
Scale
Scaling and emergent abilities
Performance scales as a power law with compute, data, and parameters. Emergent abilities — code, chain-of-thought, analogical reasoning — appear unpredictably at scale. Nobody programmed them in; they are side effects of next-token prediction at massive scale.
Domain
Continued pretraining
Steer a base model into a new domain (medicine, law, code, new language) using the same next-token objective on domain data. Much cheaper than full pretraining. Key: train embed_tokens + lm_head too; use rsLoRA rank 256.
Data
Data quality beats size
Phi-3 (3.8B params) trained on textbook-quality data rivals 13B models on raw internet text. Llama 3 uses aggressive deduplication and quality classifiers. Data curation is now as important as model architecture.
Scaling Laws for Neural Language Models — Kaplan et al., OpenAI 2020 (arXiv:2001.08361).
Training Compute-Optimal LLMs (Chinchilla) — Hoffmann et al., DeepMind 2022.
Layer 9 — training optimizations
Engineering solutions that make large-scale training stable, fast, and memory-efficient. Every frontier training run uses all of these simultaneously.
AdamW
The de-facto standard optimizer
Momentum (1st moment) + adaptive learning rate (2nd moment) + weight decay applied directly to params. Stores 2 extra fp32 copies of all parameters → 56 GB+ for a 7B model. β₁=0.9, β₂=0.95, λ=0.1.
BF16
Mixed precision training
Forward + backward in BF16 (same exponent range as fp32, avoids overflow). Optimizer step in fp32 master weights. Halves memory and roughly doubles throughput on A100/H100. BF16 preferred over FP16 since 2022.
LR
Warmup + cosine decay
Linear warmup from ~0 to peak LR over first 1–10% of steps. Cosine decay to 1–10% of peak. GPT-3: peak 6e-4, 375M warmup tokens. Prevents early gradient explosion and late oscillation.
ZeRO
ZeRO + gradient checkpointing
ZeRO-3 shards model weights, gradients, and optimizer states across GPUs — linear memory reduction with GPU count. Gradient checkpointing recomputes activations during backward instead of storing them → √N memory at 1.33× compute.
Layer 10 — supervised fine-tuning & PEFT
Adapting the base model to follow instructions and domain tasks. LoRA and QLoRA make this feasible on consumer hardware by training only 1% of parameters.
SFT
Supervised fine-tuning
Train on (prompt, completion) pairs. Compute loss only on completions (train_on_responses_only). Teaches that questions expect answers. Formats: Alpaca, ShareGPT, ChatML. 1K–100K examples.
LoRA
Low-rank adaptation
Freeze all weights. Add thin matrices A (d×r) and B (r×d) per layer. Train only A and B. r=16 → ~0.1% of 7B params trained. At inference: merge BA into W with zero overhead. Matches full fine-tuning on most tasks.
QLoRA
4-bit quantized LoRA
Quantize frozen base model to NF4 4-bit. LoRA adapters stay in 16-bit. 70B model fits in <35 GB with QLoRA. Double quantization + paged optimizers. 39% slower than LoRA but 4× less VRAM.
PEFT
Other PEFT methods
Adapters (bottleneck modules inside blocks), Prefix tuning (virtual tokens at every layer's K&V), Prompt tuning (k soft input tokens), DoRA (magnitude + direction decomposition), rsLoRA (alpha/√r for high-rank stability).
LoRA: Low-Rank Adaptation of Large Language Models — Hu et al., Microsoft 2021 (arXiv:2106.09685).
QLoRA: Efficient Finetuning of Quantized LLMs — Dettmers et al. 2023 (arXiv:2305.14314).
Layer 11 — alignment (RLHF)
Turning a text predictor into a helpful, honest, harmless assistant. The reward model encodes human preferences; reinforcement learning optimises the policy against it.
BT
Bradley-Terry reward model
Loss: −log σ(r_w − r_l). Trains a scalar reward predictor on (prompt, chosen, rejected) triplets. Human annotators pick the preferred response. Reward model learns relative preference without needing absolute scores.
BoN
Best-of-N (rejection sampling)
Generate N responses, score with reward model, SFT on best. No RL required. Llama 2 used 4 iterative rounds before PPO. Simple and effective baseline.
PPO
PPO-KL and PPO-Clip
PPO-KL adds β×KL(π_θ‖π_ref) to the reward signal. PPO-Clip clips probability ratio to [1−ε,1+ε]. Requires 4 models simultaneously (policy, reference, reward model, critic). Used in InstructGPT/ChatGPT.
DPO
Direct preference optimisation
Skip the reward model entirely. Closed-form mapping from preference pairs to policy gradient. Trains like SFT. No RL instability. L = −E[log σ(β log(π_θ(y_w)/π_ref) − β log(π_θ(y_l)/π_ref))]. Rafailov et al. 2023.
Training language models to follow instructions (InstructGPT) — Ouyang et al., OpenAI 2022 (arXiv:2203.02155).
Direct Preference Optimization — Rafailov et al., Stanford 2023 (arXiv:2305.18290).
Layer 12 — reasoning training (RLVR)
Teaching models to generate extended chain-of-thought reasoning using verifiable rewards instead of learned reward models.
GRPO
Group relative policy optimisation
Generate G outputs per prompt (G=64). Score all with verifier. Advantage: Â_i = (r_i − mean(r)) / std(r). Clip probability ratio. KL penalty in loss. No critic model → 40–60% less VRAM than PPO.
RLVR
Verifiable rewards
Math answer correct/wrong: binary reward. Code passes unit tests: binary reward. Format reward: <think> tags present. No learned reward model needed → no reward hacking. DeepSeek-R1: 4-stage training pipeline.
Emerge
Emergent reasoning
R1-Zero (base model, no SFT, pure GRPO) spontaneously developed self-evaluation, "aha moments", backtracking on errors, and extended thinking for harder problems — without any chain-of-thought examples.
Bench
Reasoning benchmarks
AIME (15 competition math, Pass@1 avg 32 samples), MATH-500, GSM8K (saturated ≥97%), HumanEval (code), GPQA Diamond (PhD science), LiveCodeBench (fresh competitive programming).
DeepSeekMath: Pushing the Limits of Mathematical Reasoning — Shao et al. 2024 (arXiv:2402.03300). Introduced GRPO.
DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via RL — DeepSeek-AI 2025 (arXiv:2501.12948).
Layer 14 — multimodal: vision & speech
The same transformer architecture extended to images and audio by changing only the input tokenisation layer. Everything after that is identical.
ViT
Vision Transformer patches
224×224 image → 196 patches of 16×16px. Each patch flattened to 768 numbers → linear projection → D-dim embedding + learned position encoding. [CLS] token gathers global image representation for classification.
CLIP
Contrastive language-image pretraining
Train image encoder (ViT) + text encoder (GPT) so matching (image, caption) pairs have high dot product. 400M internet pairs. Zero-shot 76% ImageNet accuracy. Powers Stable Diffusion and DALL-E.
Spec
Log-Mel spectrogram
Raw audio (16,000 samples/sec) → STFT → Mel frequency bins → log-scale → 80 bins × 3,000 time frames. CNN frontend compresses to 1,500 frames. Standard transformer encoder then processes frame tokens identically to text tokens.
Whisper
Whisper enc-dec ASR
Encoder reads Mel spectrogram. Decoder generates text tokens autoregressively via cross-attention to encoder. 680K hours, 96 languages. Sizes: Tiny (39M) to Large-v3 (1.55B). WER on English clean speech: ~2.7%.
An Image is Worth 16×16 Words (ViT) — Dosovitskiy et al., Google Brain 2020 (arXiv:2010.11929).
CLIP — Radford et al., OpenAI 2021 (arXiv:2103.00020).
Robust Speech Recognition via Large-Scale Weak Supervision (Whisper) — Radford et al., OpenAI 2022 (arXiv:2212.04356).
wav2vec 2.0 — Baevski et al., Meta 2020 (arXiv:2006.11477).
Layer 15 — VAE & stable diffusion
Generative image models that combine a VAE for compression, a U-Net diffusion process for denoising, and a CLIP text encoder for guidance.
VAE
Variational autoencoder
Encoder compresses image to latent distribution (mean μ, std σ). Decoder reconstructs from sampled z = μ + σ·ε. Loss = reconstruction + KL divergence from N(0,1). Reparameterisation trick enables backpropagation through sampling.
Latent
Latent space
Images compressed 8× by VAE before diffusion. The latent space develops meaningful directions without labels — directions for smile, age, style emerge purely from training data co-occurrence. Smooth interpolation produces plausible new images.
Diffuse
Denoising diffusion
Forward: add Gaussian noise over T=1,000 steps until image is pure noise. Reverse: U-Net predicts and removes noise at each step. At inference: start from pure noise, denoise guided by CLIP text embedding via cross-attention.
SD
Stable Diffusion (latent DM)
Diffusion runs in VAE latent space (8× smaller than pixels). U-Net + CLIP text encoder + VAE decoder. Released publicly 2022 — the first high-quality open-source text-to-image model. Enabled the Midjourney and DALL-E ecosystem.
Auto-Encoding Variational Bayes (VAE) — Kingma & Welling 2013 (arXiv:1312.6114). Invented VAEs and the reparameterisation trick.
High-Resolution Image Synthesis with Latent Diffusion Models — Rombach et al., CVPR 2022 (arXiv:2112.10752). The Stable Diffusion paper.
Layer 13 — frameworks & tooling NEW
Where conceptual algorithms become running code. TRL (Transformers Reinforcement Learning) is the production library that implements everything in Layers 10–12 — SFT, reward modelling, DPO, GRPO, and PPO — in one unified, stable API. Unsloth patches TRL's GRPO kernels to achieve 90% VRAM reduction, enabling desktop-level reasoning model training.
TRL
Transformers Reinforcement Learning
Full-stack post-training library by Hugging Face. Covers the entire alignment pipeline: SFTTrainer → RewardTrainer → GRPOTrainer / PPOTrainer / DPOTrainer. Every trainer is a thin subclass of transformers.Trainer with @dataclass config. v1.0.0, April 2026. Apache 2.0.
Pipeline
Modern alignment pipeline
Pre-trained LLM → SFTTrainer (instruction following) → two paths: (1) Offline: DPOTrainer / KTO / ORPO — no reward model, trains on static preference pairs. (2) Online: RewardTrainer → GRPOTrainer / PPOTrainer — generates rollouts, verifiable rewards or reward model signal.
Unsloth
90% VRAM reduction for GRPO
Unsloth patches TRL's GRPOTrainer kernels — avoids materialising massive logit tensors, implements Standby Mode KV-Cache, supports 380K token context windows. 8B model at 20K context: 510 GB (standard) → 54 GB (Unsloth). Single 8 GB consumer GPU now sufficient for GRPO training.
Rubric
Reward design: rubrics over scalars
The practical lesson from TRL/GRPO deployments: a single correct/incorrect reward is too sparse. A rubric — stacked smaller verifiable rewards (format, accuracy, length, chain-of-thought) — gives the model richer feedback at each step and produces dramatically more stable training convergence.
vLLM
Fast rollout generation
Online RL (GRPO) requires generating thousands of completions per training step — the most expensive part. TRL integrates vLLM as the inference backend for rollout generation, achieving up to 86× faster completion generation compared to standard HuggingFace inference during online RL.
RLVR
Patience is all you need
Core insight behind RLVR: if the initial probability of the correct answer is > 0, RL will eventually find it by pruning bad paths. This is why GRPO must be applied to SFT models (not raw base models) — SFT ensures p(correct) > 0 before RL begins. RL then amplifies what is already latent.
TRL: Transformer Reinforcement Learning — von Werra et al., Hugging Face 2020–2026. v1.0.0 April 2026. Apache 2.0. github.com/huggingface/trl
Unsloth RL: Transforming LLMs Into Reasoning Models With GRPO — Unsloth AI 2025. 90% VRAM reduction. 380K context support. Desktop-level reasoning model training.
DeepSeekMath: Pushing the Limits of Mathematical Reasoning in LLMs — Shao et al. 2024 (arXiv:2402.03300). Introduced GRPO as implemented in TRL's GRPOTrainer.
