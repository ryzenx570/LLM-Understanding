Deep dive into LLM token and positional embeddings — covering byte sizes, positional encoding algorithms, corpus-based word vectors, and scale comparison.

How tokenization works
Tokens are not words — they are byte sequences
LLMs use a system called Byte-Pair Encoding (BPE). It starts with individual bytes (the smallest unit — a single character uses 1 byte in English, up to 4 bytes in other languages) and merges the most common pairs repeatedly until it has a vocabulary of ~50,000 "chunks". Common English words become 1 token. Rare or long words get split. Non-English text uses more tokens per word.

English
~1–2 bytes per token
Common English words ("the", "is", "cat") are each 1 token = 1–3 bytes. 1 token ≈ 4 characters on average. A 1,000-word essay ≈ 1,300 tokens.
Other languages
3–4 bytes per token
Chinese, Arabic, Hindi characters use 3–4 bytes each in UTF-8. A single Chinese character is often 1 token but 3 bytes. This means non-English text "costs" more tokens.
Vocabulary size
~50,000–100,000 tokens
GPT-2 had 50,257 tokens. GPT-4 uses ~100,256. Each token gets a unique ID (an integer), which is what the model actually sees — not the text itself.
Why not just bytes?
Too many steps, too little meaning
Working with raw bytes makes sequences very long — "hello" = 5 bytes = 5 steps. BPE compresses frequent patterns so the model sees fewer, more meaningful chunks.
Why position matters
Transformers read all tokens at once — so they need position labels
Unlike old models that read left-to-right, transformers process every token simultaneously. This is faster, but it means the model has no idea which token came first. Positional encoding adds a unique "position fingerprint" to each token's embedding so the model can tell word 1 from word 500.
Sinusoidal — Vaswani et al. 2017
Sine and cosine waves at different frequencies
Each position gets a unique pattern of sine and cosine values across the embedding dimensions. Low dimensions encode coarse position, high dimensions encode fine position — like a clock with many hands at different speeds. No training needed; it just works.
Learned — GPT-2, BERT
Position treated like a vocabulary
Position 1 gets its own trainable vector, position 2 gets another, and so on up to max context length. Simple and effective, but can't generalize beyond the training length. GPT-2 learned 1,024 position vectors.
RoPE — Llama, Mistral, GPT-NeoX
Rotation in embedding space
Rotary Position Embedding rotates the query and key vectors in attention by an angle proportional to their position. The relative distance between two tokens is preserved naturally — so the model can generalize to sequences longer than it was trained on. Used in almost all modern open models.
ALiBi — MPT, BLOOM
Attention bias instead of embedding
Instead of adding position to the embeddings, ALiBi subtracts a penalty from attention scores based on how far apart two tokens are. Farther apart = less attention. Elegant, no extra parameters, and generalizes well to long contexts.
Where do word meanings come from?
Embeddings are learned from billions of sentences in a corpus
A corpus is the giant collection of text the model trains on — Wikipedia, books, web pages, code. The model learns that "king" and "queen" appear in similar contexts, so they get similar embedding vectors. No human labels the meanings — the structure emerges from co-occurrence patterns alone.

Word2Vec — Mikolov et al. 2013
The breakthrough embedding paper
Trained a shallow network to predict a word from its neighbors (CBOW) or neighbors from a word (Skip-gram). Result: vectors where king − man + woman ≈ queen. Trained on ~100 billion words. Changed NLP forever.
GloVe — Pennington et al. 2014
Global co-occurrence statistics
Instead of local context windows, GloVe built a giant matrix of how often every word appeared with every other word across the whole corpus, then factored it. More stable training, similar quality to Word2Vec.
Contextual embeddings — BERT, GPT
One word, many meanings
"Bank" near "river" gets a different vector than "bank" near "money". Modern LLMs produce a new embedding for every word in every sentence — context changes the vector every time. This is why they understand ambiguity.
Embedding dimension
Each word = 768 to 12,288 numbers
GPT-2 small: 768 dimensions. GPT-3: 12,288 dimensions. Each number captures a tiny aspect of meaning — not interpretable individually, but together they encode grammar, semantics, world knowledge, and style.
Training an embedding layer vs training a full LLM
The difference is staggering — orders of magnitude
An embedding layer is a lookup table: one vector per vocabulary item. Training it is fast, cheap, and runs on a laptop. A full LLM stacks 96 transformer blocks on top and requires thousands of specialized GPUs running for months.
