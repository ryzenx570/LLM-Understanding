LLM Training Pipeline: Pretraining, Training Optimizations, Supervised Fine-Tuning, and Parameter-Efficient Fine-Tuning — based on Daniel Han's LLM training pipeline video.

Stage 1 of the LLM pipeline
Pretraining — teaching a model everything from raw text
Pretraining is the most expensive and foundational step. The model is trained on trillions of tokens of text using a single objective: predict the next token. No human labels are needed — the text itself is the supervision signal. This phase creates the "base model" — a powerful text predictor that has implicitly learned grammar, facts, reasoning, and world knowledge, but is not yet useful as an assistant.
Data curation — the most critical step
Garbage in, garbage out
Data quality determines model quality more than almost any other factor. Modern pretraining pipelines apply aggressive filtering: deduplication (same text appearing 100× should not dominate), quality classifiers (filtering out low-quality web pages), domain balancing (mixing code, books, Wikipedia, conversations in deliberate ratios), and toxicity filters. Llama 3 used 15 trillion tokens; GPT-3 used ~500B tokens.
Training objective — next-token prediction
One simple loss, trillion-scale consequences
The model sees a sequence of tokens and must predict each next token from all previous ones. The loss is cross-entropy between predicted probabilities and the actual next token. This single objective — applied at massive scale — causes the model to implicitly learn grammar, facts, syntax, reasoning chains, code patterns, and more. No human labels are ever needed.
Scaling laws — Chinchilla (Hoffmann et al. 2022)
How much compute, how many tokens, how big a model?
The Chinchilla paper showed that most LLMs before 2022 were overtrained on model size and undertrained on data. The optimal ratio is roughly 20 tokens per parameter — a 7B model should see ~140B tokens for compute-optimal training. Llama 3 deliberately violated this by training on 15T tokens (over-training for inference efficiency — a smaller model that trains longer can be served more cheaply at scale even if it costs more to train).

Continued pretraining — domain adaptation
Steering a base model into a new domain without full retraining
After initial pretraining on general text, a model can be continually pretrained on domain-specific data — medical literature, legal documents, code, a new language — using the same next-token prediction objective. This is much cheaper than full pretraining and lets organizations adapt public base models to their domain. Key insight from Unsloth research: train on ALL linear layers including embed_tokens and lm_head, use Rank-Stabilized LoRA (rsLoRA) at rank 256, and use different learning rates for embeddings to stabilize training.

Key pretraining papers for further study
Scaling Laws for Neural Language Models — Kaplan et al., OpenAI 2020 (arXiv:2001.08361). Showed performance scales as a power law with compute, data, and model size.

Training Compute-Optimal Large Language Models (Chinchilla) — Hoffmann et al., DeepMind 2022. Showed most models were undertrained on data; established the 20 tokens/parameter rule.

The Llama 3 Herd of Models — Meta 2024 (arXiv:2407.21783). Detailed pretraining at 15T tokens with aggressive data curation and deliberate over-training for deployment efficiency.

Stage 2 — from base model to assistant
Supervised Fine-Tuning (SFT) — teaching the model to follow instructions
A base model trained only on next-token prediction will complete your text, not answer your question. If you ask "What is the capital of France?", it might respond "What is the capital of Germany?" — because that's the kind of text it has seen. SFT teaches the model that a question expects a specific type of answer. Using thousands to hundreds of thousands of (prompt, completion) pairs, the model is trained to follow instructions, answer questions, hold conversations, and behave helpfully.
Dataset formats
What instruction data looks like
Data comes as (prompt, completion) pairs. Common formats: Alpaca (instruction + input + output fields), ShareGPT (multi-turn conversations), ChatML (system + user + assistant turns). Quality matters far more than quantity — 1,000 carefully curated examples often outperform 100,000 noisy ones. The FineTome-100k dataset combines conversations, reasoning chains, function calls, and code problems.
Train on completions only
Don't compute loss on the prompt
A key SFT trick: compute the cross-entropy loss ONLY on the assistant's response tokens, not on the user's prompt. This teaches the model to generate good answers rather than to reproduce questions. The QLoRA paper showed this significantly improves accuracy, especially for multi-turn conversations. In Unsloth this is called "train_on_responses_only".
Full fine-tuning vs LoRA vs QLoRA
The three SFT strategies — and when to use each
Instruction tuning vs standard SFT
Fewer examples, better generalization
Standard SFT needs thousands of task-specific examples. Instruction tuning uses (prompt, completion) pairs that demonstrate behavior — "always be helpful, honest, and harmless" — rather than task-specific answers. Models learn to generalize the behavior pattern, not memorize answers. InstructGPT showed a 1.3B instruction-tuned model outperformed a raw 175B GPT-3 on helpfulness.
Catastrophic forgetting
The danger of full fine-tuning
When you update all weights during SFT, the model can "forget" general abilities learned during pretraining — a phenomenon called catastrophic forgetting. Fine-tuning a coding model might make it lose conversational ability. PEFT methods (LoRA, adapters) mitigate this by keeping base weights frozen and only training small additions, preserving the original knowledge.
Paper annotations
Key SFT papers
Training language models to follow instructions with human feedback (InstructGPT) — Ouyang et al., OpenAI 2022 (arXiv:2203.02155). Introduced RLHF for instruction following. Showed smaller instruction-tuned models beat larger raw models.

Self-Instruct: Aligning Language Models with Self-Generated Instructions — Wang et al. 2022 (arXiv:2212.10560). Showed LLMs can generate their own instruction data, democratizing SFT dataset creation.

QLoRA: Efficient Finetuning of Quantized LLMs — Dettmers et al. 2023 (arXiv:2305.14314). Enabled fine-tuning of 65B models on a single 48GB GPU. Introduced 4-bit NormalFloat quantization and double quantization.
Parameter-Efficient Fine-Tuning
Fine-tune a 70B model on your laptop — by only training 1% of its weights
PEFT is a family of techniques that adapt a large pretrained model to a new task by training only a tiny fraction of its parameters, while keeping the base model frozen. The result: you get most of the benefit of full fine-tuning at 1–10% of the compute and memory cost. This has democratized LLM adaptation — researchers and developers can now fine-tune state-of-the-art models on consumer hardware.

LoRA — how the low-rank decomposition works
Where LoRA adapters are inserted
Target modules in the transformer
LoRA adapters are injected into selected linear weight matrices. Common targets: Q, K, V, O (attention projections), up_proj, down_proj, gate_proj (feed-forward layers). Training ALL linear layers including embed_tokens and lm_head produces better results, especially for continued pretraining. Unsloth defaults to all linear layers for comprehensive adaptation.
LoRA hyperparameters
The key knobs to tune
r (rank) — controls capacity. r=8 minimal, r=16 standard, r=64–256 for continued pretraining. alpha — scaling factor, typically set to r or 2r. dropout — 0.05–0.1 to prevent overfitting. learning rate — 2e-4 for standard SFT. If training loss falls below 0.2, the model is overfitting. rsLoRA uses alpha/√r scaling for stability at high ranks.
Paper annotations — PEFT
Key PEFT papers for further study
LoRA: Low-Rank Adaptation of Large Language Models — Hu et al., Microsoft 2021 (arXiv:2106.09685). Introduced LoRA. Showed it matches full fine-tuning on GPT-3 tasks while training only 0.01% of parameters.

QLoRA: Efficient Finetuning of Quantized LLMs — Dettmers et al. 2023 (arXiv:2305.14314). Combined LoRA with 4-bit NF4 quantization. Enabled 65B model fine-tuning on a single 48GB GPU. Introduced paged optimizers and double quantization.

DoRA: Weight-Decomposed Low-Rank Adaptation — Liu et al. 2024 (arXiv:2402.09353). Decomposes weights into magnitude and direction components, training both separately. Often outperforms LoRA with similar compute.

Parameter-Efficient Transfer Learning for NLP — Houlsby et al., Google 2019 (arXiv:1902.00751). Original adapters paper — inserted small bottleneck modules into transformer layers. Foundational PEFT work predating LoRA.
