RLHF Alignment and Reasoning Training: PPO variants, DPO, Best-of-N, Bradley-Terry, GRPO, advantage, loss, and evaluation benchmarks for reasoning LLMs.

Why alignment matters
A base model is not an assistant — it just completes text
After pretraining, a model knows how to predict the next token but has no concept of "being helpful" or "avoiding harm." Ask it a question and it may continue with more questions. Ask for help and it may output training text patterns. Reinforcement Learning from Human Feedback (RLHF) is the family of techniques that bridge this gap — teaching the model what humans actually want from it.
Stage 2 — The reward model
A neural network that predicts human preference
Human annotators are shown pairs of responses (A vs B) to the same prompt and pick which they prefer. A reward model is then trained on these (prompt, chosen, rejected) triplets to predict which response a human would prefer — assigning a scalar score. The reward model is usually a separate model of similar or smaller size to the LLM being aligned, with a linear head on top of a transformer.
Bradley-Terry formulation
The probability model behind preference learning
The reward model is trained using the Bradley-Terry pairwise comparison model. If response y_w is preferred over y_l, the probability of that preference is modeled as σ(r(x,y_w) − r(x,y_l)), where σ is sigmoid. The loss is: −log σ(r_w − r_l). This elegant formula says: push the reward of the chosen response higher than the rejected one. No absolute score needed — only relative preference.
On-policy vs off-policy alignment
Two fundamentally different approaches to using preference data
On-policy (PPO, GRPO): The model generates new responses during training. The reward model scores them. The RL algorithm updates the model's weights. This is expensive — inference is slow — but achieves the highest performance ceiling because the model always trains on its own current behavior.

Off-policy (DPO, ORPO): Training uses pre-collected preference data without generating new responses during training. Simpler, faster, more stable — but the model never explores beyond the fixed dataset. Less adaptive than on-policy methods for complex reasoning tasks.
Proximal Policy Optimization
The dominant on-policy RL algorithm for LLM alignment
PPO was introduced by OpenAI in 2017 (Schulman et al.) and became the backbone of ChatGPT's alignment process (InstructGPT). In the LLM setting, the "policy" is the LLM, "actions" are tokens, "states" are the current generation so far, and the "reward" comes from the reward model at the end of each complete response.

Four models required during PPO training
PPO objective — combined loss
The full PPO objective has three terms:
1. Clipped surrogate — maximize advantage while staying close to old policy (clip at ε = 0.2).
2. KL penalty — subtract β × KL(π_θ ‖ π_ref) to prevent reward hacking.
3. Value loss — train the critic to predict future rewards accurately.
Typical β = 0.02–0.1. Higher β = more conservative.
PPO limitations for LLMs
Memory: 4 models in GPU at once — policy, reference, reward, critic. For a 7B model this is ~80GB+ VRAM.
Stability: Balancing policy + critic training simultaneously is notoriously tricky.
Speed: Must generate completions on-policy (slow inference) before each update.
Complexity: Many hyperparameters (β, ε, GAE λ, clip ratio) to tune simultaneously.
Best-of-N (Rejection Sampling)
The simplest alignment strategy — generate many, keep the best
Generate N responses from the model for each prompt, score all N with the reward model, keep only the highest-scoring one as training data, then fine-tune the model on these "best" responses via SFT. No RL required. Llama 2's alignment used four rounds of rejection sampling before any RL. Simple and effective for moderate alignment goals, but computationally wasteful (most generations are discarded) and doesn't explore beyond the current model's distribution.
Direct Preference Optimization (DPO)
Skip the reward model entirely — optimize directly on preference pairs
DPO (Rafailov et al., Stanford 2023) made a key mathematical observation: the optimal RLHF policy has a closed-form solution that can be derived directly from the preference data — no separate reward model required. By rearranging the RLHF objective, DPO shows that the reward is implicitly encoded in the ratio of log-probabilities between the policy and a reference model.
DPO advantages
No reward model to train and maintain. No RL instability. Trains like standard supervised fine-tuning — simple cross-entropy-style loss. Memory efficient — only 2 models needed (policy + frozen reference). More stable and reproducible than PPO. Used widely in open-source models.
DPO limitations
Off-policy — trains on fixed preference data, never explores new responses. Cannot handle tasks requiring complex reasoning chains where correctness is hard to define as a pairwise preference. Quality depends entirely on the preference dataset. Generally considered weaker than on-policy RL for hard reasoning tasks like math and code.
Paper annotations
Direct Preference Optimization: Your Language Model is Secretly a Reward Model — Rafailov et al., Stanford 2023 (arXiv:2305.18290). Introduced DPO. Showed the optimal RLHF policy can be derived in closed form. One of the most cited alignment papers of 2023.

Scaling LLM Test-Time Compute Optimally — Snell et al. 2024. Showed Best-of-N with a strong verifier can outperform larger models at test time, even without any training.
Reasoning LLMs — a new training paradigm
Teaching models to "think before they answer"
Standard RLHF teaches models to be helpful and safe. Reasoning training — Reinforcement Learning from Verifiable Rewards (RLVR) — teaches models to solve hard problems by generating long chain-of-thought reasoning traces. The key difference: rewards come from verifiable answers (math results, code tests) rather than from a learned preference model. DeepSeek-R1 showed that pure RL on verifiable rewards can produce models that "think" step by step without any supervised chain-of-thought data.
What changes from standard RLHF
Reward source: Instead of a trained reward model (which can be "hacked"), rewards come from verifiable ground truth — the math answer is correct or not, the code compiles and passes tests or not. Binary. Objective. Unhackable.

Format rewards: Additional rewards for using the correct thinking format (e.g., reasoning inside <think> tags, answer inside <answer> tags).

Emergent behaviors: Models trained this way spontaneously develop self-correction, "aha moments," extended deliberation, and backtracking — behaviors not present in SFT data.
DeepSeek-R1-Zero — the unexpected breakthrough
Skip SFT entirely — apply RL directly to base model
DeepSeek's most surprising finding: starting from a base model (DeepSeek V3) with NO SFT phase, pure GRPO training on verifiable rewards produced emergent reasoning behavior. The model learned to allocate more thinking time to harder problems, self-evaluate answers, and backtrack on errors — all without being shown any chain-of-thought examples. Language consistency rewards were later added to prevent "language mixing" (switching between Chinese and English mid-thought).

Loss function for reasoning training
RLVR loss — maximise expected verifiable reward with KL constraint
The core objective: J(θ) = E[R(x,y)] − β · KL(π_θ ‖ π_ref)

Where R(x,y) is the verifiable reward (1 if answer correct, 0 otherwise for binary; or a continuous score from a process reward model). The KL term prevents the model from drifting too far from the reference policy. In GRPO, this is computed per-token across the generated sequence, and the advantage normalises rewards within a group of completions for the same prompt.
Group Relative Policy Optimization
PPO without the critic — DeepSeek's key innovation
GRPO was introduced in DeepSeekMath (Shao et al. 2024, arXiv:2402.03300) and later used to train DeepSeek-R1. The key insight: instead of training a separate critic (value function) to estimate the baseline for advantage calculation, GRPO estimates the baseline from the average reward of a group of responses generated for the same prompt. Eliminates one entire model from PPO's 4-model setup.
PPO — 4 models
Policy (trained) + Reference (frozen) + Reward model (frozen) + Critic / value function (trained)

The critic predicts "how much total reward will this state lead to?" — a hard function to learn, especially for text. Requires ~56GB+ VRAM for a 7B model.
GRPO — 3 models (no critic)
Policy (trained) + Reference (frozen) + Reward model (frozen)

The baseline is estimated from the group of responses themselves — no learned value function. Saves 40–60% VRAM. Up to 18× more cost-efficient than PPO in DeepSeek's experiments.
GRPO step by step
Why group sampling works as a baseline
A value function (critic) must learn to predict future reward from an intermediate state — notoriously hard for text. GRPO sidesteps this by observing multiple complete trajectories for the same prompt. The group average reward is a Monte Carlo estimate of the expected reward — simpler, more direct, and needs no separate network. The key requirement: generate enough samples per prompt (G ≥ 16, typically 64) for the estimate to be stable.
Reward hacking prevention
Two mechanisms prevent the model from gaming the reward:
1. KL penalty — directly in the loss function, penalizes divergence from the reference policy. More conservative updates.
2. Relative advantage — comparing within a group makes the model focus on relative quality rather than absolute reward scores. Exploiting the reward model would require all outputs to be poor, which the clip prevents.
Evaluating reasoning models
How we measure whether a model can actually think — click each benchmark
Reasoning benchmarks test multi-step problem solving, not pattern matching. The key metrics are Pass@k (does at least one of k attempts succeed?), majority voting (most common answer from N samples), and exact match on verifiable answers. Frontier models have saturated many old benchmarks — the field continuously pushes to harder problems.

Evaluation metrics for reasoning models
Pass@k
Generate k independent responses. Pass if at least one is correct. Pass@1 = accuracy. Pass@100 = ceiling of model's potential. DeepSeek-R1 is evaluated with Pass@1 using 32 samples (averaged for AIME).
Majority voting (self-consistency)
Generate N responses (e.g., 32), return the most common answer. Much more reliable than a single sample, especially for hard math. Often boosts AIME scores by 10–20 percentage points over Pass@1.
Exact match
Compare model answer to ground truth exactly. Used for math (numeric answers) and multiple-choice. For code, unit tests replace exact match — the code must run and produce correct outputs.
Benchmark contamination
A critical risk: if training data contains benchmark questions, scores are inflated. A 2023 study found removing contaminated examples dropped GSM8K scores by up to 13% for some models. Always check contamination disclosures.
