Animated explainer: how a text token becomes a vector, based on 3Blue1Brown's transformer series

Step 1 — Tokenisation. The sentence is split into tokens. Each token gets a unique integer ID. "cat" → token #4758. "The" and "the" both map to #262 — the same token!
Step 2 — The lookup table. The model keeps a giant matrix — one row per vocabulary item. "cat" (ID #4758) fetches its row: a list of 12,288 floating-point numbers. This is the raw embedding — the token's starting position in meaning-space.
Step 3 — The vector as a bar chart. Each dimension is one bar. "cat" and "dog" have very similar bar patterns because they appear in similar contexts during training. "cat" and "democracy" would look completely different.
Step 4 — Geometry of meaning. In the high-dimensional space, directions encode concepts. The "gender" direction points from male words to female words. Subtract it from "king", add it back — and you land near "queen". The model learned this from co-occurrence alone — never told what gender means.
Step 5 — Context reshapes the vector. The word "bank" starts as the same vector both times. After the attention block reads surrounding words, the vector shifts — pulled toward "water" in one sentence, toward "money" in the other. This is what makes contextual embeddings powerful.
Full picture — token to prediction. Text → integer IDs → static vectors (lookup) → position added → attention reshapes with context → next word predicted. The embedding layer is just the start — 0.35% of GPT-3's parameters, but the essential first step.
