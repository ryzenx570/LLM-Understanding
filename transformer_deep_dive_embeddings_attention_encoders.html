The evolution of position encoding
From hardcoded sine waves to learned rotations — click each era to expand
Transformers process all tokens simultaneously — they have no built-in sense of order. Position encodings solve this by injecting "where am I in the sequence?" information into each token's vector. The technique has evolved dramatically from a fixed mathematical formula in 2017 to dynamic rotations used in every frontier model today.
2017
Hardcoded sinusoidal encoding — the original
Vaswani et al. "Attention Is All You Need" · used in: original Transformer
Each token at position p gets a fixed vector of sine and cosine values at different frequencies. No parameters are learned — the encoding is computed once from a formula and added directly to the embedding. Low-frequency dimensions encode coarse position ("am I near the start or end?"), high-frequency dimensions encode fine position ("exactly which word am I?"). Think of it like a clock with many hands spinning at different speeds — every moment in time produces a unique combination of hand positions.

Formula: PE(p, 2i) = sin(p / 10000^(2i/d)) and PE(p, 2i+1) = cos(p / 10000^(2i/d)) where d is the embedding dimension.

Weakness: Fixed frequencies mean the model cannot adapt position encoding to the data. Performance degrades on sequences longer than seen during training because the sine patterns repeat or extrapolate poorly.
2018
Learned absolute position embeddings
Used in: BERT, GPT-1, GPT-2
Instead of a fixed formula, each position index (1, 2, 3 … max_length) gets its own learnable embedding vector — just like a word in the vocabulary. These vectors are initialized randomly and updated during training alongside all other model parameters.

How it works: Add a position lookup table alongside the token lookup table. Token at position 5 gets token_embed[token_id] + position_embed[5]. Both tables are learned jointly.

GPT-2 example: 1,024 position vectors × 768 dimensions = ~786K additional parameters. Small relative to the total model size.

Strength: The model freely learns whatever positional patterns are most useful for the data, rather than following a prescribed formula.

Weakness: Hard ceiling at the trained maximum length. BERT trained with 512 positions — it simply cannot handle longer sequences without retraining or tricks. The position 513 vector does not exist.
2019
Relative position encodings
Shaw et al. 2018, used in: Transformer-XL, T5
Instead of encoding "I am at position 47", relative encodings ask "how far apart are these two tokens?" When computing attention between token A and token B, a learned bias term is added based on (position_B − position_A), not on their absolute positions.

Key insight: For many NLP tasks, what matters is not whether a word is at position 50 vs position 100, but whether it is 3 words away from another word. Relative encoding captures this directly.

T5's implementation: Rather than continuous distance values, T5 buckets distances into 32 logarithmically spaced bins. Far-away tokens all share the same bucket — the model only needs fine-grained distance info for nearby tokens.

Advantage: Much better generalization to longer sequences — the model has seen "2 tokens apart" millions of times even if it has never seen absolute position 800.

Weakness: More computationally complex to implement. Adds parameters to every attention layer.
2021
ALiBi — Attention with Linear Biases
Press et al. 2021 · used in: BLOOM, MPT
ALiBi skips adding anything to the embeddings entirely. Instead, it modifies the attention scores directly — subtracting a penalty proportional to the distance between two tokens before the softmax.

How it works: attention_score(i, j) = Q·K − m × |i − j| where m is a per-head slope (a fixed geometric sequence across heads). Farther apart = bigger penalty = less attention. No extra parameters.

Key advantage: Generalizes to sequences much longer than training length. Because it is a simple linear penalty, it degrades gracefully — tokens 2,000 positions apart get a large penalty but not infinite, so the model still attends to them a little.

BLOOM: The 176-billion-parameter multilingual model from the BigScience consortium used ALiBi as its position scheme, trained on 46 languages simultaneously.
2022
RoPE — Rotary Position Embedding
Su et al. 2021 (arXiv:2104.09864) · used in: Llama 1/2/3, Mistral, Falcon, GPT-NeoX, Gemma, Phi, Claude (reportedly)
RoPE is now the dominant position encoding in almost every state-of-the-art open model. Rather than adding a position vector to the embedding, RoPE rotates the query (Q) and key (K) vectors in the attention computation by an angle proportional to the token's position.

The core intuition: If you rotate two vectors by angles θ₁ and θ₂, their dot product (used in attention) naturally depends on the difference θ₂ − θ₁ — which encodes relative distance. So RoPE gives relative position information "for free" inside the attention operation itself.

Concretely: The embedding is split into pairs of dimensions. Each pair is treated as a 2D coordinate. At position p, the pair is rotated by angle p × θ where θ is a frequency (different for each pair of dimensions, following a geometric sequence like the original sine encoding). The rotation matrix is: [cos(pθ) −sin(pθ) / sin(pθ) cos(pθ)].

Why it won:
• Naturally encodes relative distances (Q·K depends only on position difference)
• Works with FlashAttention and other efficient attention kernels
• Decays gracefully with distance — far-apart tokens automatically attend less
• Can be extended beyond training length with YaRN, LongRoPE, and other tricks

Llama 3: Uses RoPE with a base frequency of 500,000 (vs 10,000 in the original Llama), enabling a 128K token context window.

Paper: "RoFormer: Enhanced Transformer with Rotary Position Embedding" — Su, Lu, Pan, Murtadha, Wen, Liu · 2021
2023+
Extended RoPE — YaRN, LongRoPE, NTK-aware scaling
Used in: Llama 3 (128K), Mistral v0.3, Phi-3, GPT-4 Turbo (128K)
RoPE by itself struggles when asked to attend to positions much longer than seen during training — the angles "wrap around" and distant tokens look similar to nearby ones. Several techniques were developed to extend RoPE beyond its training range without full retraining.

NTK-aware scaling: Rescale the frequency base (e.g. 10,000 → 500,000) so rotation angles grow more slowly, making longer sequences representable without wrapping.

YaRN (Yet another RoPE extensioN): Applies different scaling factors to different frequency dimensions — low-frequency dimensions (coarse position) get little scaling, high-frequency ones (fine position) get heavy scaling. More precise than uniform scaling.

LongRoPE: Microsoft's approach that fine-tunes the model on a small set of long documents after applying NTK scaling. Used in Phi-3 to extend from 4K → 128K context.

Real-world impact: Llama 3 uses base frequency 500,000 vs Llama 2's 10,000 — a 50× increase — enabling genuine 128K context windows. This is the difference between reading a short article and reading an entire book in a single prompt.
At a glance — all five approaches
The attention scaling problem
Standard attention is quadratic — it does not scale to long contexts
In standard full attention, every token attends to every other token. A sequence of N tokens requires N × N attention scores to be computed and stored. Double the sequence length, quadruple the compute and memory. For a 128K-token context window, full attention would require computing 128,000 × 128,000 = 16.4 billion scores — per layer, per forward pass. This is where sliding window attention and other sparse patterns come in.
Attention patterns — step through each method
Full attention (1 of 4)
Sliding window attention — Beltagy et al. 2020, Mistral 2023
Each token only attends to its W nearest neighbors
Instead of attending to all N tokens, each token only attends to the W tokens immediately before it (the "window"). This reduces the attention computation from O(N²) to O(N × W) — linear in sequence length if W is fixed. Information still propagates across long distances because each layer passes context forward: layer 1 covers 1 window, layer 2 sees information from 2 windows, and so on. After L layers, each token has an "effective receptive field" of L × W tokens.
Longformer (2020) — Beltagy et al.
Local + global attention
Most tokens use sliding window attention (local). A small number of "global" tokens (like [CLS]) attend to all tokens and are attended by all tokens. Used in long-document tasks. Reduced memory from O(N²) to O(N × W). First practical demonstration that sliding window attention works on real NLP tasks up to 4,096 tokens.
Mistral 7B (2023) — sliding window in a decoder
Window size 4,096, rolling KV cache
Mistral applied sliding window attention to a decoder-only LLM, combined with a "rolling buffer" KV cache — old key-value pairs beyond the window are discarded. Each of Mistral's 32 layers uses a window of 4,096 tokens, giving an effective receptive field of 32 × 4,096 = 131,072 tokens — a 128K effective context using a fixed-size cache.
Grouped Query Attention — GQA
Share key-value heads across query heads
Standard multi-head attention has H separate Q, K, and V heads. GQA groups multiple Q heads to share a single K and V head. Llama 3 uses 8 KV heads shared across 32 Q heads — an 8× reduction in KV cache memory. Faster inference, less memory, nearly identical quality. Now standard in most new models.
FlashAttention (2022) — Dao et al.
IO-aware exact attention — faster, less memory
FlashAttention does not change which tokens attend to which — it computes exactly the same result as full attention but in a memory-efficient way by tiling the computation across GPU memory blocks. Combined with sliding window patterns, it makes long-context attention practical. FlashAttention-2 is now used by almost every frontier model.

Encoder-family models

While decoder-only models dominate generation tasks, encoder-based models remain the backbone of search engines, classification systems, and structured text transformation. BERT, T5, and BART represent three different philosophies of what a transformer encoder is for.

Training objective — Masked Language Modeling (MLM)
BERT randomly masks 15% of tokens and trains the model to predict the masked words from context. Unlike GPT (which only uses left context), BERT reads both left and right simultaneously — hence "bidirectional". This gives BERT a much richer understanding of each word in context, but it cannot generate new text because it has no causal structure.
Second training objective — Next Sentence Prediction (NSP)
BERT also trains on sentence pairs: given two sentences A and B, predict whether B actually follows A in the original document (50% of the time it does, 50% it is a random sentence). This teaches BERT to understand relationships between sentences — useful for question answering and inference tasks. Later work (RoBERTa) found NSP to be mostly unnecessary and dropped it.
Special tokens
[CLS] is added at the start of every input — its final hidden state is used as a "sentence representation" for classification tasks. [SEP] separates two input segments (e.g. a question and a passage). [MASK] replaces tokens during MLM training. These special tokens are part of BERT's vocabulary and have learned embeddings.
Real-world use
BERT powers Google Search's understanding of queries. It is used in sentiment classifiers, spam filters, named entity recognition, and extractive question answering (finding the answer span within a document). RoBERTa (2019) improved BERT by training longer with more data and dropping NSP. DeBERTa (2020) adds disentangled attention separating content and position. ALBERT (2020) reduces parameters by sharing weights across layers.

Core philosophy — everything is text-to-text
T5's defining idea is that every NLP task can be framed as converting one text string into another. Translation: "translate English to French: The cat sat on the mat" → "Le chat était assis sur le tapis". Summarization: "summarize: [long article]" → "[short summary]". Classification: "sentiment: I love this movie" → "positive". This unified framing lets T5 use a single model and training setup for dozens of tasks.
Architecture details
T5 uses the full encoder-decoder design from the original 2017 transformer. The encoder reads the input with full bidirectional attention. The decoder generates the output autoregressively with masked self-attention and cross-attention to the encoder. T5 uses relative position biases (32 logarithmically spaced buckets) instead of learned absolute positions — better generalization to longer sequences. Layer normalization is applied before each sub-layer (pre-norm), not after (post-norm), which stabilizes training.
Variants and successors
mT5 extends T5 to 101 languages for multilingual tasks. Flan-T5 (2022) fine-tunes T5 on 1,836 tasks with instructions, dramatically improving zero-shot performance. UL2 unifies multiple pre-training objectives. T5 remains one of the most widely deployed models in enterprise NLP pipelines — its text-to-text framing made it trivially adaptable to any task without architecture changes.

Core idea — corrupt text, then reconstruct it
BART's key insight is to think of pre-training as a denoising problem. The encoder reads a corrupted version of the text (with tokens deleted, shuffled, masked, or sentences permuted). The decoder then generates the original, uncorrupted text. This forces the encoder to build a deep understanding of what the text should be, and the decoder to generate fluent text — making BART excellent at any task involving reconstruction or generation from a representation.
Five corruption strategies used during pre-training
Token masking — like BERT, randomly replace tokens with [MASK].
Token deletion — remove tokens entirely (the model must also figure out where they were).
Text infilling — replace a span of text with a single [MASK] token (variable length spans → single mask).
Sentence permutation — shuffle the order of sentences in the document.
Document rotation — rotate the document so it starts from a random token (the model must identify the true start).

Text infilling proved most effective and is the dominant objective. It requires the model to predict how many tokens are missing — a harder task than BERT's one-to-one masking.
BART vs BERT vs T5
BERT is encoder-only — it understands but cannot generate. T5 is encoder-decoder — it generates by framing everything as text-to-text with a single corruption objective (span masking). BART is encoder-decoder — it generates using five different corruption strategies, making it more robust. BART's decoder is a full autoregressive language model trained from the start to reconstruct text, which gives it stronger generation abilities than T5 on tasks like abstractive summarization and dialogue. mBART extends BART to 25 languages, and mBART-50 covers 50 languages for multilingual translation.
