Layer 0 — input & data
Before a single weight is updated, raw text must be collected, cleaned, and converted into integer token IDs. Everything the model knows is determined by this layer.
Corpus
Data curation
Deduplication, quality filtering (fastText classifiers), domain balancing. Llama 3: 15T tokens. GPT-3: 500B tokens. Data quality beats data quantity at every scale.
BPE
Byte-pair encoding tokenisation
Merges most-frequent byte pairs until vocab size is reached. 50K–100K tokens. "cat" → 1 token. "neurotransmitter" → 4 tokens. 1–4 bytes per token depending on language and script.
Bytes
Token byte sizes
ASCII English: 1–3 bytes/token. Chinese characters: 3 bytes each (UTF-8). Emoji: 4 bytes. Numbers often split digit-by-digit — the core reason LLMs struggle with arithmetic.
Scale
Chinchilla scaling laws
Hoffmann et al. 2022: optimal compute = ~20 tokens per parameter. Llama 3 deliberately over-trains for inference efficiency. GPT-3 was heavily undertrained by Chinchilla standards.
Scaling Laws for Neural Language Models — Kaplan et al., OpenAI 2020 (arXiv:2001.08361). Power-law relationship between compute, data, model size, and loss.
Training Compute-Optimal LLMs (Chinchilla) — Hoffmann et al., DeepMind 2022. 20 tokens/param rule. Showed most 2021 models were massively undertrained on data.
The Llama 3 Herd of Models — Meta 2024 (arXiv:2407.21783). 15T tokens, aggressive curation pipeline, deliberate over-training for deployment efficiency.
Layer 1 — token + positional embedding
Integer token IDs are looked up in a giant table to retrieve dense vectors. Positional information is injected so the transformer knows word order.
Lookup
Embedding table
One row per vocabulary item. Each row = a vector of 768 (GPT-2) to 12,288 (GPT-3) floats. GPT-3: 50,257 × 12,288 = 617M parameters just in the embedding layer.
Geometry
Vector as meaning
Similar words cluster nearby in vector space. king − man + woman ≈ queen. Context reshapes meaning through attention — "bank" near "river" vs "bank" near "loan" produces different vectors.
Position
Positional encoding evolution
Sinusoidal (2017) → Learned absolute (BERT/GPT-2) → Relative/T5 buckets → ALiBi (linear bias) → RoPE (rotates Q&K, dominant since 2022) → YaRN / LongRoPE (500K+ context).
RoPE
Rotary position embedding
Rotates Q and K vectors by angle proportional to position. Q·K dot product naturally encodes relative distance. Used in Llama 1/2/3, Mistral, Falcon, Gemma. Llama 3 base freq 500,000 → 128K context.
RoFormer: Enhanced Transformer with Rotary Position Embedding — Su et al. 2021 (arXiv:2104.09864). Introduced RoPE. Now the standard positional encoding for frontier open-weight models.
Layer 2 — embedding algorithms
How the embedding vectors in Layer 1 are trained. A progression from shallow count-based methods in the 1990s to deep contextual transformers today.
CBOW
Continuous bag of words
Average context word vectors → predict the missing centre word. Many→one. Fast training, good for frequent words. Deep parallel: BERT's MLM is CBOW with a full transformer instead of a shallow average.
Skip-gram
Skip-gram + negative sampling
One word → predict all surrounding words. One→many. 4× more training signals per position. Excellent for rare words. Negative sampling updates only 5–20 wrong words per step — 1000× speedup.
GloVe
Global vectors (Stanford 2014)
Factorises the full corpus co-occurrence matrix globally instead of using local windows. More stable training. Paris:France ≈ Berlin:Germany emerges from the global matrix structure.
FastText
Subword embeddings (Meta 2017)
Breaks words into character n-grams. Can represent unseen words. Essential for morphologically rich languages like Turkish, Finnish, and Arabic where the same root generates hundreds of word forms.
Efficient Estimation of Word Representations in Vector Space (Word2Vec) — Mikolov et al., Google 2013 (arXiv:1301.3781). Introduced CBOW and Skip-gram in a single paper.
GloVe: Global Vectors for Word Representation — Pennington, Socher, Manning, Stanford 2014. Global co-occurrence matrix factorisation.
Layer 3 — attention mechanism
The heart of the transformer. Every token computes relationships with every other token via query, key, and value projections. Attention is what makes transformers powerful.
QKV
Scaled dot-product attention
Q·Kᵀ / √d_k gives alignment scores. Softmax normalises to probabilities. Multiply by V for weighted sum of values. Multi-head: run in parallel with different learned projections to capture different relationship types.
Mask
Causal mask (decoder)
Upper-triangle filled with −∞ prevents attending to future tokens. After softmax, −∞ → 0 exactly. Trained on all positions in one forward pass — massively efficient. At inference each new token reruns full attention.
Window
Sliding window attention
Each token attends only to W nearest neighbours → O(N·W) instead of O(N²). Mistral 7B: window=4,096 × 32 layers = 131K effective context. Rolling KV cache discards old key-value pairs automatically.
GQA
Grouped query attention
Multiple Q heads share one K+V head. Llama 3: 32 Q heads, 8 KV heads → 4× smaller KV cache at inference. FlashAttention computes the same result as full attention but tiles it on GPU for 2–4× memory savings.
Attention Is All You Need — Vaswani et al., Google Brain 2017 (arXiv:1706.03762). 90K+ citations. Introduced the transformer, multi-head attention, and scaled dot-product attention.
FlashAttention-2 — Dao et al. 2022/2023. IO-aware exact attention. Standard in nearly every frontier model training run for efficient long-context training.
Layer 4 — transformer block internals
After attention, each token passes through a feed-forward network. Layer normalisation and residual connections stabilise training across many stacked blocks.
FFN
Feed-forward network
Two linear layers with GELU activation between them, 4× wider than embedding dim. Applied to each token independently — no cross-token communication. Believed to store most of the model's factual knowledge.
Norm
Layer normalisation
Pre-norm (applied before each sublayer) is now standard — more stable than original post-norm. Rescales activations to mean 0, std 1. Prevents exploding/vanishing gradients across N stacked blocks.
Skip
Residual connections
Output = LayerNorm(input) + sublayer(input). The identity path bypasses each sublayer. Makes gradients flow directly from output to early layers — the critical ingredient enabling 96-layer depth in GPT-3.
Stack
Stacking blocks × N
GPT-2: 12 blocks. GPT-3: 96 blocks. Lower blocks learn syntax and surface patterns. Upper blocks encode reasoning, facts, and abstract concepts. Each block: ~0.18× params in attention, ~0.72× in FFN.
Layer 5 — architecture families
How many transformer blocks, in what configuration, with what training objective — these decisions define whether you get BERT, T5, or GPT.
Enc-only
BERT · RoBERTa · DeBERTa
Bidirectional attention — no causal mask. MLM + NSP pretraining. [CLS] for classification, [MASK] for prediction, [SEP] for segment boundary. Cannot generate text. Powers Google Search, classification, NER.
Enc-dec
T5 · BART · mT5
T5: text-to-text framing for every task — classification, translation, summarisation all become "generate this output". BART: 5 corruption strategies. Cross-attention connects encoder to decoder.
Dec-only
GPT · Llama · Claude · Gemini
Causal mask only. No encoder, no cross-attention. One objective: next-token prediction. Scales cleanly to trillions of parameters. All frontier assistants use decoder-only architecture + RLHF alignment on top.
Why
Why decoder-only dominates
One simple loss scales to any text. No encoder-decoder split needed. The prompt IS the context — the decoder attends to it causally. RLHF converts the next-token predictor into an assistant without any architecture change.
BERT: Pre-training of Deep Bidirectional Transformers — Devlin et al., Google 2018 (arXiv:1810.04805). Masked language modelling + next sentence prediction. Foundation of the BERT family.
Exploring the Limits of Transfer Learning (T5) — Raffel et al., Google 2019 (arXiv:1910.10683). Text-to-text framework unifying all NLP tasks in one model.
BART: Denoising Sequence-to-Sequence Pre-training — Lewis et al., Meta 2019 (arXiv:1910.13461). Five corruption strategies; state of the art for summarisation.
Layer 6 — output head
After N transformer blocks, the final hidden state is projected to vocabulary logits and a probability is sampled for the next token. This closes the forward pass.
Logits
Linear projection
A weight matrix maps the D-dimensional hidden state to V-dimensional logits (one per vocab item). GPT-3: 12,288 → 50,257. This matrix shares weights with the embedding table (weight tying) in modern models.
Prob
Softmax + sampling
Softmax converts logits to probabilities summing to 1. Temperature <1 = sharper (more confident). Temperature >1 = flatter (more creative). Top-p: sample from smallest set of tokens summing to p (e.g. 0.9).
Loss
Cross-entropy loss
−log(probability assigned to correct next token). Averaged across all positions. Perplexity = exp(loss) — the standard language model metric. Lower is better. GPT-2 on Wikipedia: ~35. GPT-4: ~5.
Decode
Decoding strategies
Greedy: always pick the highest-probability token. Beam search: keep top-k sequences at each step. Temperature + top-p for creative generation. Beam search used in Whisper for accurate speech transcription.
Layer 7 — pretraining
Training the base model from scratch on trillions of tokens. The most expensive stage — but everything downstream depends on the quality of this foundation.
Objective
Next-token prediction
Predict each next token given all previous ones. One loss, any text, no labels required. At scale, this simple objective causes the model to learn grammar, facts, world knowledge, coding, and reasoning as side effects.
Scale
Scaling and emergent abilities
Performance scales as a power law with compute, data, and parameters. Emergent abilities — code, chain-of-thought, analogical reasoning — appear unpredictably at scale. Nobody programmed them in; they are side effects of next-token prediction.
Domain
Continued pretraining
Steer a base model into a new domain (medicine, law, code, new language) using the same next-token objective on domain data. Much cheaper than full pretraining. Key: train embed_tokens + lm_head too; use rsLoRA rank 256.
Data
Data quality beats size
Phi-3 (3.8B params) trained on textbook-quality data rivals 13B models on raw internet text. Llama 3 uses aggressive deduplication and quality classifiers. Data curation is now as important as model architecture.
Scaling Laws for Neural Language Models — Kaplan et al., OpenAI 2020 (arXiv:2001.08361).
Training Compute-Optimal LLMs (Chinchilla) — Hoffmann et al., DeepMind 2022.
The Llama 3 Herd of Models — Meta 2024 (arXiv:2407.21783).
Layer 8 — training optimizations
Engineering solutions that make large-scale training stable, fast, and memory-efficient. Every frontier training run uses all of these simultaneously.
AdamW
The de-facto standard optimizer
Momentum (1st moment) + adaptive learning rate (2nd moment) + weight decay applied directly to params. Stores 2 extra fp32 copies of all parameters → 56GB+ for a 7B model. β₁=0.9, β₂=0.95, λ=0.1.
BF16
Mixed precision training
Forward + backward in BF16 (same exponent range as fp32, avoids overflow). Optimizer step in fp32 master weights. Halves memory and roughly doubles throughput on A100/H100. BF16 preferred over FP16 since 2022.
LR
Warmup + cosine decay
Linear warmup from ~0 to peak LR over first 1–10% of steps. Cosine decay to 1–10% of peak. GPT-3: peak 6e-4, 375M warmup tokens. Prevents early gradient explosion and late oscillation around the optimum.
ZeRO
ZeRO + gradient checkpointing
ZeRO-3 shards model weights, gradients, and optimizer states across GPUs — linear memory reduction with GPU count. Gradient checkpointing recomputes activations during backward instead of storing them → √N memory at 1.33× compute.
Layer 9 — supervised fine-tuning & PEFT
Adapting the base model to follow instructions and domain tasks. LoRA and QLoRA make this feasible on consumer hardware by training only 1% of parameters.
SFT
Supervised fine-tuning
Train on (prompt, completion) pairs. Compute loss only on completions (train_on_responses_only). Teaches that questions expect answers. Formats: Alpaca, ShareGPT, ChatML. 1K–100K examples. InstructGPT: 1.3B SFT+RLHF beats 175B base.
LoRA
Low-rank adaptation
Freeze all weights. Add thin matrices A (d×r) and B (r×d) per layer. Train only A and B. r=16 → ~0.1% of 7B params trained. At inference: merge BA into W with zero overhead. Matches full fine-tuning on most tasks.
QLoRA
4-bit quantized LoRA
Quantize frozen base model to NF4 4-bit. LoRA adapters stay in 16-bit. 70B model fits in <35GB with QLoRA vs 140GB full precision. Double quantization + paged optimizers. 39% slower than LoRA but 4× less VRAM.
PEFT
Other PEFT methods
Adapters (bottleneck modules inside blocks), Prefix tuning (virtual tokens at every layer's K&V), Prompt tuning (k soft input tokens), DoRA (magnitude + direction decomposition), rsLoRA (alpha/√r for high-rank stability).
LoRA: Low-Rank Adaptation of Large Language Models — Hu et al., Microsoft 2021 (arXiv:2106.09685). Trains 0.01% of GPT-3 parameters; matches full fine-tuning quality.
QLoRA: Efficient Finetuning of Quantized LLMs — Dettmers et al. 2023 (arXiv:2305.14314). 65B model fine-tuning on a single 48GB GPU. NF4 + double quantization + paged optimizers.
Layer 10 — alignment (RLHF)
Turning a text predictor into a helpful, honest, harmless assistant. The reward model encodes human preferences; reinforcement learning optimises the policy against it.
BT
Bradley-Terry reward model
Loss: −log σ(r_w − r_l). Trains a scalar reward predictor on (prompt, chosen, rejected) triplets. Human annotators pick the preferred response. Reward model learns relative preference without needing absolute scores.
BoN
Best-of-N (rejection sampling)
Generate N responses, score with reward model, SFT on best. No RL required. Llama 2 used 4 iterative rounds before PPO. Simple and effective baseline. Wasteful at large N since most generations are discarded.
PPO
PPO-KL and PPO-Clip
PPO-KL adds β×KL(π_θ‖π_ref) to the reward signal. PPO-Clip clips probability ratio to [1−ε,1+ε]. Requires 4 models simultaneously (policy, reference, reward model, critic). Very memory intensive — used in InstructGPT/ChatGPT.
DPO
Direct preference optimisation
Skip the reward model entirely. Closed-form mapping from preference pairs to policy gradient. L = −E[log σ(β log(π_θ(y_w)/π_ref) − β log(π_θ(y_l)/π_ref))]. Trains like SFT. No RL instability. Rafailov et al. Stanford 2023.
Training language models to follow instructions (InstructGPT) — Ouyang et al., OpenAI 2022 (arXiv:2203.02155). RLHF for chat models. Foundation of ChatGPT.
Direct Preference Optimization — Rafailov et al., Stanford 2023 (arXiv:2305.18290). Eliminates the reward model. One of the most cited alignment papers of 2023.
Layer 11 — reasoning training (RLVR)
Teaching models to generate extended chain-of-thought reasoning using verifiable rewards instead of learned reward models. The approach that produced DeepSeek-R1.
GRPO
Group relative policy optimisation
Generate G outputs per prompt (G=64 in DeepSeekMath). Score all with verifier. Advantage: Â_i = (r_i − mean(r)) / std(r). Clip probability ratio. KL penalty in loss. No critic model → 40–60% less VRAM than PPO.
RLVR
Verifiable rewards
Math answer correct/wrong: binary reward. Code passes unit tests: binary reward. Format reward: <think> tags present. No learned reward model needed → no reward hacking possible. DeepSeek-R1 used a 4-stage training pipeline.
Emerge
Emergent reasoning
R1-Zero (base model, no SFT, pure GRPO) spontaneously developed self-evaluation, "aha moments", backtracking on errors, and extended thinking for harder problems. None of these behaviours were in training examples.
Bench
Reasoning benchmarks
AIME (15 competition math, Pass@1 averaged over 32 samples), MATH-500, GSM8K (saturated ≥97%), HumanEval (code), GPQA Diamond (PhD science), LiveCodeBench (fresh competitive programming). Pass@k and majority voting as metrics.
DeepSeekMath: Pushing the Limits of Mathematical Reasoning — Shao et al. 2024 (arXiv:2402.03300). Introduced GRPO. First demonstration that RL without SFT improves mathematical reasoning.
DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via RL — DeepSeek-AI 2025 (arXiv:2501.12948). R1-Zero: pure RL on base model. R1: 4-stage pipeline with open weights.
Layer 12 — multimodal: vision & speech
The same transformer architecture extended to images and audio by changing only the input tokenisation layer. Everything after that is identical.
ViT
Vision Transformer patches
224×224 image → 196 patches of 16×16px. Each patch flattened to 768 numbers → linear projection → D-dim embedding + learned position encoding. [CLS] token gathers global image representation for classification.
CLIP
Contrastive language-image pretraining
Train image encoder (ViT) + text encoder (GPT) so matching (image, caption) pairs have high dot product. 400M internet pairs. Zero-shot classification: 76% ImageNet accuracy, no fine-tuning. Powers Stable Diffusion and DALL-E.
Spec
Log-Mel spectrogram
Raw audio (16,000 samples/sec) → STFT → Mel frequency bins → log-scale → 80 bins × 3,000 time frames. CNN frontend compresses to 1,500 frames. Standard transformer encoder then processes frame tokens identically to text tokens.
Whisper
Whisper enc-dec ASR
Encoder reads Mel spectrogram. Decoder generates text tokens autoregressively via cross-attention to encoder. 680K hours, 96 languages. Sizes: Tiny (39M) to Large-v3 (1.55B). WER on English clean speech: ~2.7%.
An Image is Worth 16×16 Words (ViT) — Dosovitskiy et al., Google Brain 2020 (arXiv:2010.11929). Showed pure transformer outperforms CNNs at scale.
CLIP — Radford et al., OpenAI 2021 (arXiv:2103.00020). Foundation of all major text-to-image models.
Robust Speech Recognition via Large-Scale Weak Supervision (Whisper) — Radford et al., OpenAI 2022 (arXiv:2212.04356).
wav2vec 2.0 — Baevski et al., Meta 2020 (arXiv:2006.11477). ASR with just 10 minutes of labeled data via self-supervised pretraining.
Layer 13 — VAE & stable diffusion
Generative image models that combine a VAE for compression, a U-Net diffusion process for denoising, and a CLIP text encoder for guidance.
VAE
Variational autoencoder
Encoder compresses image to latent distribution (mean μ, std σ). Decoder reconstructs from sampled z = μ + σ·ε. Loss = reconstruction + KL divergence from N(0,1). Reparameterisation trick enables backpropagation through sampling.
Latent
Latent space
Images compressed 8× by VAE before diffusion. The latent space develops meaningful directions without labels — directions for smile, age, style emerge purely from training data co-occurrence. Smooth interpolation produces plausible new images.
Diffuse
Denoising diffusion
Forward: add Gaussian noise over T=1,000 steps until image is pure noise. Reverse: U-Net predicts and removes noise at each step. At inference: start from pure noise, denoise guided by CLIP text embedding via cross-attention.
SD
Stable Diffusion (latent DM)
Diffusion runs in VAE latent space (8× smaller than pixels). U-Net + CLIP text encoder + VAE decoder. Released publicly 2022 — the first high-quality open-source text-to-image model. Enabled the Midjourney and DALL-E ecosystem.
Auto-Encoding Variational Bayes (VAE) — Kingma & Welling 2013 (arXiv:1312.6114). Invented VAEs and the reparameterisation trick. Foundation of modern generative image models.
High-Resolution Image Synthesis with Latent Diffusion Models — Rombach et al., CVPR 2022 (arXiv:2112.10752). The Stable Diffusion paper. Made HD generation feasible on consumer GPUs.
